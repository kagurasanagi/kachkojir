## micro SDカードについて
SPIでもSDIOでも接続できるようなピンレイアウトで接続する。
SDIO接続のほうが早いのでSDIO前提で作成していく。
### プルアップ
CMD,DATA0-3は10kΩで外部プルアップしている。10kΩ～100kΩ推奨なので下限ギリギリより22kΩのがいいらしいが10kΩの抵抗たくさん持ってるので10kΩいれた。
### GNDを遠回りさせない
GNDはベタで太くやること。
ブレッドボードで実験するにしても、GNDのラインを電源近くから複数とらないとデータが若干化けてCRCエラーが出ることがあった。
### 厚めの電源供給
電源はrp2040ボードの3.3vなどからとらずに別途用意したLDOからとること。500mAあれば大丈夫かと。
rp2040と共通電源ラインのNJM12877F33（200ｍA）からSDカードにつないでいたら、データとしてF(=16)が期待されるところでFだけ欠けるという現象に遭遇して1日無駄にした。
電源供給周りは将来的に2次電池での動作を考えたときに問題になる可能性がある。
SPIでの低速通信のほうが要求される電源ピークや通信線の質が低いため、バッテリー動作時はSPI,電源供給時はSDIOで接続されるようにプログラムを組もう。
消費電流のボラティリティが高いので0.1uF+1uFのセラミックコンデンサ2つと、100uFの導電性高分子アルミ固体電解コンデンサーを実装する。
### SDカード電源ライン用LDO
TLV755Pあたりを想定。（500mA、200uFまでOK）
### 終端抵抗
CLKには終端抵抗として47Ω入れている。これがないと高速で動かしたときにCRCエラーが頻発する。
CMD,DATA0-3にも入れたほうがいいという話を聞くが、CLKほど重要ではないようなので入れなくていい。
これらの条件で、ブレッドボード上でもbaud_rate=30MHzでSDカードへの読み書きが行えた。
